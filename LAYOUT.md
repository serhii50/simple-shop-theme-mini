# Спецификация и API Темы (Theme Design Contract)

Этот документ описывает строгий архитектурный контракт между **"Движком"** (Simple Shop Core) и любой **"Темой"** (presentation layer), которая к нему подключается. 

Проект концептуально разделен на две части:
1. **Движок (Core):** Хранит базу товаров (`_data/products.csv`), настройки (`_config.yml`), SEO данные и плагины для генерации файлов.
2. **Тема (Theme):** Отвечает исключительно за генерацию HTML-каркаса, CSS-стилизацию (инструмент/фреймворк выбирается автором темы) и клиентский JavaScript (корзина). Тема **не** должна содержать товаров движка или пытаться напрямую изменить конфигурацию сервера (Jekyll build).

> [!NOTE]
> При создании или редактировании шаблонов в этом проекте, ИИ и разработчики должны строго придерживаться описанных ниже контрактов глобальных переменных и структуры данных.

## 1. Глобальные переменные среды (От Движка к Теме)
Любая тема при сборке страниц может рассчитывать на следующие глобальные объекты конфигурации, которые движок гарантированно передаст:

* `site.title` / `site.description` — базовые метатеги (используются в SEO).
* `site.currency` — Валюта магазина (например, "грн." или "руб."). Должна интегрироваться везде рядом с ценами.
* `site.whatsapp_phone` — Номер телефона (из `_config.yml`) для отправки заказа в WhatsApp API.
* `site.telegram_user` — Юзернейм (из `_config.yml`) для формирования заказа в Telegram API.

## 2. Модель Данных Товара (Model)
Товары магазина хранятся в CSV-базе `_data/products.csv` без подключения бэкенда. Тема, которая производит цикл по каталогу (например, на главной странице `{% for item in site.data.products %}`), получает доступ к следующим ключам:

* `item.id` (Уникальный числовой артикул)
* `item.title` (Название для отображения текста)
* `item.slug` (URL-совместимый идентификатор, транслитерация названия. Используется в ссылках на товар: `/product/SLUG/`)
* `item.price` (Число)
* `item.description` (Многострочное описание)
* `item.images` (Строка со ссылками на фото. Несколько фото разделяются символом `;`. Тема обязана разбить их фильтром: `{% assign images = item.images | split: ';' %}`)
* `item.tags` (Строка с тегами товара. Несколько тегов разделяются символом `;`. Поле необязательное — может быть пустым. Тема обязана разбить фильтром: `{% assign tags = item.tags | split: ';' %}` и проверить наличие перед выводом.)

> [!IMPORTANT]
> Поле для названия товара — `title`, а **не** `name`. Встроенный метод `Page#name` в Jekyll возвращает имя файла (например, `2.html`), перекрывая данные из CSV. Поэтому в CSV используется колонка `title`, которая корректно доступна через `page.title`.

## 3. Требования к Входным Точкам (Layouts)
Любая тема, подключаемая к **Simple Shop**, обязана иметь минимум следующие файлы макетов, чтобы не сломать генерацию системных файлов и продуктовых страниц.

### 3.1 `_layouts/default.html`
Основной каркас для всего сайта. 
* Обязан содержать открывающие и закрывающие архитектурные элементы: `<html>`, `<head>`, `<body>`.
* Обязан содержать переменный тег `{{ content }}`, куда будет подставляться основное содержимое.
* Обязан содержать `{{ custom_head }}` внутри `<head>` — для инъекции OG-метатегов с дочерних страниц.

### 3.2 `_layouts/home.html`
Каталог товаров (главная страница).
* Карточки товаров: карусель миниатюр (если у товара несколько фото), кликабельные теги (фильтруют каталог), SVG-иконка корзины.
* Фильтрация: панель тегов вверху каталога, при клике на тег в карточке — прокрутка и активация фильтра.

### 3.3 `_layouts/product.html`
Шаблон для генерации детальной карточки продукта (генерируется системно плагином `page_gen`). 
* В отличие от главной страницы (где идет итерация `item`), плагин page_gen загружает текущий товар в глобальную переменную **`page`**.
* Доступ к переменным на этой странице осуществляется так: `page.title`, `page.price`, `page.images` (которые так же нужно сплитить по `;`).
* Обязан содержать: галерею изображений (основное фото + миниатюры), теги товара, OG-метатеги (`og:title`, `og:image`, `og:description`) для корректных превью в мессенджерах.

### 3.4 `_layouts/cart.html`
Корзина покупателя.

## 4. Контракт Корзины (The Cart Flow)
Simple Shop не использует серверную базу данных, сессии или куки (MySQL/PostgreSQL, PHP) для покупок. Вся логика магазина — **исключительно клиентская**. Любая спроектированная тема обязана соответствовать следующим стандартам:

### Хранилище (Local Storage)
* Ключ базы данных браузера: `mySimpleCart`
* Формат записи: массив JSON объектов (например: `[{id, title, price, image, qty}]`).

### Страница Оформления (Checkout)
В теме обязательно должна присутствовать логика корзины (по умолчанию располагается по URL `/cart`).
* **Функция:** Считывает массив из `localStorage`.
* **Задача:** Собирает текстовое представление заказа (формирует список: товар — цена × кол-во, считает итоговую сумму) и трансформирует в ссылки мессенджеров:
  - `https://wa.me/ТЕЛЕФОН?text=ТЕКСТ_ЗАКАЗА`
  - `https://t.me/ЮЗЕРНЕЙМ?text=ТЕКСТ_ЗАКАЗА`
* Сообщение должно включать **ссылки на каждый товар** (`/product/ID.html`) для возможности просмотра превью.
* Поведение: Тема **не очищает** корзину после переадресации в мессенджер — покупатель должен иметь возможность вернуться и повторно оформить заказ.

## 5. Специфика Стилизации (CSS/JS)
Тема `simple-shop-theme-mini` использует **исключительно нативные технологии**:
* **CSS:** Только Vanilla CSS (без фреймворков и препроцессоров — никакого Tailwind, Bootstrap, SCSS)
* **JavaScript:** Только Vanilla JS (без библиотек — никакого jQuery, Alpine.js и т.д.)

Это принципиальное ограничение темы, обеспечивающее нулевую зависимость от внешних пакетов.
